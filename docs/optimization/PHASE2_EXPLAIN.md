# Phase 2: 统一常量优化 — 解释

## 一、什么是统一常量优化？

### 1.1 问题描述

**当前问题**：同一个常量（如牌宽）在多个地方定义，数值不一致。

```python
# animations.py 中的问题
def reveal_card(...):
    CARD_W = 36  # 第244行

def card_back_raw():
    w = 34  # 第322行

def question_raw():
    w = 34  # 第363行

def text_card_raw():
    w = 36  # 第402行
```

### 1.2 为什么这是问题？

| 问题 | 影响 |
|------|------|
| **对齐不一致** | 不同动画的牌宽度不同，视觉上不对齐 |
| **修改困难** | 改牌宽要改 4 处，容易遗漏 |
| **bug 风险** | 改一处忘记改其他处，导致显示异常 |
| **可读性差** | 不知道 34/36 代表什么含义 |

### 1.3 统一常量的好处

| 好处 | 说明 |
|------|------|
| **一致性** | 所有地方用同一个值，保证对齐 |
| **易维护** | 改一处生效，不会遗漏 |
| **可读性** | 常量名有意义，如 `CARD_WIDTH` |
| **可测试** | 可以测试常量值是否正确 |

## 二、当前常量使用情况

### 2.1 animations.py 中的牌宽

```python
# 第244行 — reveal_card
CARD_W = 36

# 第322行 — card_back_raw
w = 34

# 第363行 — question_raw
w = 34

# 第402行 — text_card_raw
w = 36
```

**问题**：34 和 36 混用，可能导致对齐问题。

### 2.2 其他潜在常量

| 常量 | 当前状态 | 问题 |
|------|---------|------|
| 牌宽 | 34/36 混用 | 对齐不一致 |
| 牌高 | 未统一 | 可能有类似问题 |
| 动画速度 | 硬编码 | 难以调整 |
| 颜色 | 已统一到 style.py | ✅ 良好 |

## 三、优化方案

### 3.1 创建统一常量模块

在 `style.py` 中添加：

```python
# 卡片尺寸常量
CARD_WIDTH = 36
CARD_HEIGHT = 12

# 动画参数
ANIMATION_SPEED = 0.05
SHUFFLE_DURATION = 2.0
```

### 3.2 替换硬编码

```python
# animations.py — 使用统一常量
from tarot.style import CARD_WIDTH, CARD_HEIGHT

def card_back_raw():
    w = CARD_WIDTH  # 而不是硬编码 34
    ...

def reveal_card(...):
    CARD_W = CARD_WIDTH  # 而不是硬编码 36
    ...
```

### 3.3 修改一处生效

```python
# style.py — 只改这里
CARD_WIDTH = 38  # 从 36 改为 38

# 所有引用的地方自动生效：
# - animations.py
# - renderer.py
# - app.py
```
