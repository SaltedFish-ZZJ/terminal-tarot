# Phase 3: 代码质量优化计划

## 优化任务清单

### 🔴 P0 - 高优先级（核心风险）

#### 1. SpreadUI 核心业务测试
- **目标**: 为 `spreads_ui.py` 的三个 reading 方法添加单元测试
- **风险**: 这些是核心业务流程，无测试覆盖是最大风险
- **方法**: `three_card_reading()`, `celtic_cross_reading()`, `love_spread_reading()`
- **预估**: 需要 mock 数据和 UI 组件

#### 2. pytest-cov 覆盖率集成
- **目标**: 量化测试质量，目标 >80% 覆盖率
- **收益**: 识别未覆盖的关键代码路径
- **配置**: pyproject.toml 中添加 coverage 配置

### 🟡 P1 - 中优先级（维护性）

#### 3. deck.py 数据外部化
- **目标**: 将 588 行硬编码数据改为 JSON/YAML
- **收益**: 数据与逻辑分离，便于维护和扩展
- **范围**: 大阿卡纳、小阿卡纳数据

#### 4. 添加 ruff linting
- **目标**: 自动代码质量检查
- **配置**: pyproject.toml 中配置 ruff 规则
- **收益**: 统一代码风格，提前发现潜在问题

#### 5. 添加 mypy 类型检查
- **目标**: 静态类型安全
- **收益**: IDE 补全增强，提前发现类型错误
- **配置**: pyproject.toml 中配置 mypy

### 🟢 P2 - 低优先级（代码整洁）

#### 6. style.py 拆分
- **目标**: 颜色定义 vs 工具函数分离
- **收益**: 更清晰的代码组织

#### 7. 统一输出方式
- **目标**: 消除 app.py 中 Rich 和 sys.stdout.write 混用
- **收益**: 一致的输出体验

#### 8. 添加 __all__ 导出
- **目标**: 明确公共 API
- **收益**: 防止意外暴露内部实现
